    Set Play Calendar Implementation Testing

    Sep 22, 20244 min read

    Background

    Thinking through the idea Set play day and how I could test it out in my current The Way I Work v2. If useful I’ll keep if not I’ll throw it away.

    For now I’ll think through how I would actually do it. Testing this out and it’s affects comes later.

    Flow of thought

    • How should I store set play?

      • Does this need to be automated? Should I build this out manual prove set play is useful then reach for tools to help the side effects of extra capacity to use?
        • yeah
        • What is the least effort way to do it
          • What does it need to do?
            • Requirements
              • define A day with set blocks of time
              • Define a week?
                • I don’t’ know if I have the cadence of a week template really needed lets rule out of scope focus on day usecase
              • Tagging what practices that template accomplishes
                • This would help me find templates that would give me chess and other tasks needing to be done
              • How do I manage template bloat?
                • will selecting a template just be more work than planning that day?
                  • The idea is that it could help with batching
                  • Will need to just keep an eye on this
            • Options that I see
              • Notion
                • Copy pasting from a calendar database or maybe even it’s templates being able to have subtasks that are relative to it’s parent so that it schedules.
                • Could see then if google cal sync solves long term auto if needed
                • Practices would need to be duplicated from Obsidian documentation for The Way I Work v2 and to be also tagged in notion.
              • Obisidian?
                • notes with links to the practices they accomplish - that allows be to filter them to active practices
                  • Should make a tag to deprecate practices I leave behind
                • This would mean my plans are in markdown format.. That might work but that might make automation harder long term if needed
            • I should test out obsidian
              • It can be hard to get a sense of time in markdown to see what is too crowded of a day or whether it would fit with standing meeting that is on the day already
                • Might explore the Full Calendar Obsidian Plugin to fix that.
                  • I don’t know if it can do templates but it can be a calendar in the vault
                  • Might look for how Time Ruler Obsidian Plugin plays with these as well
                    • Tested this out too cumbersome doesn’t solve any of my above issues.
                  • I could use Full Calendar Obsidian Plugin and Templater Obsidian Plugin to make these tempaltes
                    • This looks promising
                    • Testing this out I am able to make a templater template that gives me the ability to inject the template into the calendar
                      • Now I need to test Calendar in Obsidian → google calendar or similar
                      • There is a bug in the full calendar implementation fo the EventStore for calendar events. It’s primary key is the event id coming from the ical source. These id’s are not unique in the case of adding multiple ical sources where one calendar may share an event with another. Thus they are failing to import due to unique key constraint.
                        • Household calendar and personal calendar are the cause of the conflict in this case
                    • Another issue is tagging in this
                      • The tags get applied when using the template need to test removal prior to template use
                      • Testing using tags in a templater comment to see if obisidian will pick it up for search but templater won’t propagate it
                        • Works. using <%**%> and embedding the tags inside of that.

    • When I pick a set play how do I get it on the calendar?

      • Potential plugin for browser to do the setting
      • Could also just write using the API
        • API seems reasonable but won’t do it until the process shows value

    • How do I make sure that I’m doing my chess or other activities at a good cadence if I’m relying on the occurrence of a set play?

      • My Tracker setup will play this role.

    Outcome

    • Use my Daily Plan practice to pick a set play.
    • Use Templater Obsidian Plugin to stamp out the events for that set play
      • Can tag the template with the practices these set plays accomplish
    • Use Full Calendar Obsidian Plugin to view these events on calendar view to rationalize time visually
    • Google Calendar remains source of truth for executing the day’s plan
